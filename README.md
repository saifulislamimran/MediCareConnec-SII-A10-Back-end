The robust, highly secure RESTful API powering the MediCare application. Built with Node.js, Express, and MongoDB, this backend enforces strict data integrity, handles complex aggregations for pagination, and acts as the sole authority for payment intents and Role-Based Access Control (RBAC).

🔗 **[Live Frontend Demo](https://medi-care-connec-sii-a10-front-end.vercel.app)** | 🔗 **[Live Backend API](https://medi-care-connec-sii-a10-back-end.vercel.app)**

## 🛡️ Enterprise-Grade Security & Architecture

*   **Server-Side Price Authority:** Stripe Payment Intents are generated by securely fetching the consultation fee directly from MongoDB, completely neutralizing client-side payload manipulation risks.
*   **Concurrency & Overbooking Control:** The Appointment Controller implements strict time-slot validation to prevent double-booking.
*   **Database Seeding Mechanism:** Includes a hidden `/api/seed/doctors` route to automatically populate the database with professional, mock doctor profiles.
*   **Bulletproof JWT Authentication:** Deeply integrated authentication middleware (`verifyToken`, `verifyRole`) that intercepts unauthorized API requests before they reach the controllers.
*   **Optimized Data Hoarding Prevention:** Implements server-side pagination (`skip`, `limit`) and filtering to prevent bandwidth overload on the client.

## 📂 Core API Routing Structure

| Route Path | Responsibility | Access Level |
| :--- | :--- | :--- |
| `/api/auth/*` | Registration, Login, Token generation. | Public |
| `/api/appointments/*` | Booking creation, conflict resolution. | Patient, Admin |
| `/api/doctor-portal/*` | Doctor-specific dashboard management. | Doctor, Admin |
| `/api/admin/*` | Global user and system management. | Admin Only |
| `/api/payments/*` | Stripe Intent generation via server-side pricing. | Patient |
| `/api/seed/*` | Automated Database Seeding. | Developer |

## ⚙️ Environment Variables

Create a `.env` file in the root directory:

| Variable | Description |

| `MONGO_URI` | Your MongoDB Atlas connection string. |
| `JWT_SECRET` | Super secure secret key for signing tokens. |
| `STRIPE_SECRET_KEY` | Your Stripe Secret Key for intent generation. |
| `FRONTEND_URL` | Permitted CORS origin (your Vercel frontend). |
